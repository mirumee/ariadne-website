# Agents.md

This file provides guidance to AI agents when working with code in this repository.

## Repository structure

This repo is the source for [ariadnegraphql.org](https://ariadnegraphql.org). It contains:

- `website/` — Docusaurus site (TypeScript, React)
- `ariadne/` — git submodule; its `docs/` folder is the **server** documentation source
- `ariadne-codegen/` — git submodule; its `docs/` folder is the **client** documentation source

The submodules must be initialised before the site can build:

```bash
git submodule update --init --recursive
```

## Development commands

All npm commands run from `website/`:

```bash
npm install          # install dependencies
npm run start        # dev server with hot reload (localhost:3000)
npm run build        # production static build → website/build/
npm run serve        # serve the previously-built build/ directory
npm run typecheck    # TypeScript check
```

Docker alternative (mounts `website/`, excludes `node_modules`):

```bash
docker compose up
```

## Docusaurus architecture

The site uses two separate `@docusaurus/plugin-content-docs` instances:

| Plugin id | Route base | Source path | Versioned dirs |
|-----------|-----------|-------------|----------------|
| `server` (preset) | `/server` | `../ariadne/docs` | `server_versioned_docs/`, `server_versioned_sidebars/` |
| `client` (plugin) | `/client` | `../ariadne-codegen/docs` | `client_versioned_docs/`, `client_versioned_sidebars/` |

Sidebars are auto-generated (`{ type: "autogenerated", dirName: "." }`), so directory structure drives navigation order — folders are named with numeric prefixes (e.g. `01-Docs/`, `02-Monitoring/`).

Legacy `/docs/*` URLs redirect to `/server/*` via `plugin-client-redirects`.

## Versioning workflow

Before snapshotting a version, check out the relevant tag in the submodule:

```bash
git -C ariadne fetch --tags
git -C ariadne checkout <tag>         # or ariadne-codegen for client
```

Then create the snapshot from `website/`:

```bash
npm run docusaurus docs:version:server <tag>   # server docs
npm run docusaurus docs:version:client <tag>   # client docs
```

The `/version-server-docs` Claude skill automates the server docs flow.

## Deployment

```bash
# from website/
GIT_USER=<github-username> USE_SSH=true npm run deploy
```

Deploys to `gh-pages` branch. Must be on `main` before deploying. Search is handled by Algolia DocSearch and re-indexed automatically every three days (manual trigger via the Algolia crawler dashboard).
